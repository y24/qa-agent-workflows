# Severity and Priority Criteria

## 使い分け

| 項目 | 意味 |
|---|---|
| 重要度 | 問題がテスト品質、欠陥見逃し、判定信頼性に与える影響 |
| 優先度 | いつ修正すべきか。後続作業への影響、修正容易性、波及範囲も加味する |

重要度と優先度は一致しない場合がある。表記ゆれは重要度Lowでも、全テンプレートへ波及する場合は優先度Mediumになり得る。

## 重要度

| 重要度 | 基準 | 例 |
|---|---|---|
| Critical | テスト結果の信頼性を大きく損なう | 主要機能の未確認、重大仕様の未反映、大量ケースが判定不能 |
| High | 重要な欠陥を見逃す可能性が高い | 主要条件の不足、期待結果の曖昧さ、重要テストデータ不足 |
| Medium | 実行、保守、レビュー効率に影響する | 手順の分かりにくさ、粒度ばらつき、部分的重複 |
| Low | 品質向上には有効だが即時修正必須ではない | 表記ゆれ、軽微な整理、補足改善 |

## 優先度

| 優先度 | 基準 |
|---|---|
| P0 | 実行開始前に修正しないとレビューまたはテストが成立しない |
| P1 | 早期修正が必要。重要な漏れや判定不能につながる |
| P2 | 実行前または次版で修正したい。効率や保守性に影響する |
| P3 | 余力があれば対応する改善提案 |

## 判断メモ

- 根拠が強く影響が大きい指摘ほど重要度を上げる。
- 未確認事項は、確認後に重要度が変わる可能性を明記する。
- 代表例だけでなく、影響範囲が広い場合は優先度を上げる。
