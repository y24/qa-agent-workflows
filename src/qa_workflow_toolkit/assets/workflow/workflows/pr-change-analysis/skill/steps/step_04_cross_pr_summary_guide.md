# Step 4: 横断サマリ作成ガイド

## 目的

複数PRの差分レポートと分析レポートを集約し、全体の変更傾向、共通・相互依存リスク、横断的な重点テスト領域をQA向けに俯瞰できるサマリを作る。

このStepは対象PRが複数のときに実施する。1件のときは省略するか、当該PRの要約に縮退する。

## 入力

- 各PRの `<pr-slug>/diff_report.md`、`<pr-slug>/analysis_report.md`
- 必要時 `references/nonfunctional_risk_catalog.md`

## 出力

- `summary.md`
- 形式: `templates/cross_pr_summary_template.md`
- 必要に応じて `issue_log.md`

## 手順

1. 各PRの目的・変更種別・主要な変更単位を1〜数行で俯瞰表にまとめる。
2. PR間で重複・関連する変更領域、相互依存（同一モジュール、順序依存、データ移行の前後関係）を特定する。
3. 各PRの非機能リスクを束ね、横断的に効いてくる品質特性（性能、互換性、運用・移行等）を抽出する。
4. 全体としての重点テスト領域と優先度を提示し、根拠PR・リスクIDを添える。
5. PR個別では見えにくい統合リスク（合流時の競合、設定の整合、リリース順序）を記す。
6. 未取得範囲・確認事項を集約し、`issue_log.md` と整合させる。

## 判断基準

- 個別分析の単純な再掲ではなく、横断で初めて見える関係・優先度を示す。
- 重点領域には必ず根拠（PR ID、`RK-xxx`）を添える。
- リリース順序や依存に不確実性がある場合は確認事項にする。

## このStepでやらないこと

- 個別PRの差分レポート・分析レポートの作り直し
- 根拠のない統合リスクの断定

## 完了条件

- 全対象PRが俯瞰表に含まれている。
- 相互依存・統合リスク・横断的な重点テスト領域が根拠付きで整理されている。
- 未解決事項が集約されている。

## 停止

`summary.md` を出力したら停止する。
