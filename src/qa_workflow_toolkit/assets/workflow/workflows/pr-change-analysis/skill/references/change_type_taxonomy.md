# 変更種別タクソノミ

差分レポートで各変更ファイル・変更単位に付ける種別タグの定義。QAが影響範囲とテスト方針を素早く判断するための分類であり、複数該当する場合は主たるものを選び、必要なら補助タグを併記する。

## 種別一覧

| タグ | 意味 | QAが特に見る点 |
|---|---|---|
| `機能追加` | 新しい振る舞い・画面・APIの追加 | 新規挙動の正常系/異常系、既存機能との干渉 |
| `機能変更` | 既存の振る舞い・仕様の変更 | 変更前後の差、後方互換、既存テストの陳腐化 |
| `不具合修正` | バグ・障害の修正 | 修正範囲の妥当性、回帰、再発条件 |
| `リファクタ` | 振る舞いを変えない内部整理 | 挙動不変の確認、広範囲な巻き込み |
| `設定・フラグ` | 設定値、フィーチャーフラグ、環境変数の変更 | 値による分岐、環境差、既定値の影響 |
| `依存更新` | ライブラリ・パッケージ・SDKの追加/更新 | 互換性、既知の挙動変化、セキュリティ |
| `スキーマ・移行` | DBスキーマ、マイグレーション、データ移行 | 移行の前後互換、ロールバック、データ整合性 |
| `API・契約` | 公開API、インターフェース、メッセージ契約の変更 | 利用側への影響、バージョニング、互換 |
| `インフラ・基盤` | パイプライン、IaC、デプロイ、ミドル設定 | デプロイ手順、環境差、可用性 |
| `権限・認証` | 認証・認可、ロール、アクセス制御 | 権限境界、昇格、漏れ |
| `性能・並行` | 性能最適化、キャッシュ、並行・非同期処理 | 性能特性、競合、整合性 |
| `UI・表示` | 画面表示、文言、レイアウト | 表示崩れ、文言、アクセシビリティ |
| `テスト・ドキュメント` | テストコード、ドキュメントのみ | 製品挙動への影響は基本なし（要確認） |
| `自動生成・整形` | 生成物、フォーマット、リネームのみ | 影響は限定的（根拠を添える） |

## 規模感の表し方

- 変更ファイル数、追加/削除行の概算を記す（取得できた範囲で）。
- 「広く薄い」変更（多数ファイルの軽微変更）と「狭く深い」変更（少数ファイルの中核ロジック変更）を区別する。

## 注意

- タグは影響の大小を直接表さない。`自動生成・整形` でも生成元の変更があれば本体側で評価する。
- 1つのPRに複数種別が混在することは多い。変更単位 `CH-xxx` ごとにタグを付けると後段で扱いやすい。
