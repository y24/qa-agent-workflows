# 非機能リスク・観点カタログ

Step 3でリスクとテスト観点を抽出する際に、必ず一巡する非機能の品質特性カタログ。
本ワークフローの主眼は非機能観点の洗い出しにあるため、各特性について「この変更で影響しうるか」を必ず判断し、該当なしの場合もその根拠を残す。

品質特性はISO/IEC 25010の製品品質特性を参考にしつつ、PR差分から判断しやすい観点に整理している。

## 特性別の着目点とトリガー

| 品質特性 | 着目する観点 | 差分上のトリガー例 |
|---|---|---|
| 性能効率性 | 応答時間、スループット、リソース使用、N+1、ループ・クエリ | DBアクセス、ループ処理、シリアライズ、キャッシュ、バッチ |
| 互換性 | 後方互換、データ互換、API/メッセージ契約、共存 | API/スキーマ/契約変更、依存更新、設定形式変更 |
| 可用性・信頼性 | 障害時挙動、リトライ、タイムアウト、冪等性、フォールバック | 外部連携、非同期、キュー、トランザクション境界 |
| セキュリティ | 認証・認可、入力検証、機密情報、インジェクション、権限境界 | 認証/権限、入力処理、ログ出力、シークレット、依存更新 |
| 保守性・運用性 | ログ、監視、フィーチャーフラグ、設定変更の運用影響 | ログ/メトリクス変更、設定・フラグ、デプロイ手順 |
| 移行性・可搬性 | データ移行、ロールバック可否、環境差、デプロイ順序 | マイグレーション、IaC、環境変数、デプロイ構成 |
| 拡張性・スケーラビリティ | 負荷増時の挙動、状態共有、スケールアウト適合 | 並行処理、グローバル状態、ロック、セッション |
| ユーザビリティ・アクセシビリティ | 操作性、文言、エラーメッセージ、アクセシビリティ | UI/文言変更、エラーハンドリング、フォーム |
| 国際化・地域対応 | 文字コード、ロケール、タイムゾーン、通貨・書式 | 日時/数値/通貨処理、文言、エンコーディング |
| 法令・データ保護 | 個人情報、監査ログ、保持期間、同意 | 個人データ処理、ログ、外部送信 |

## 使い方

1. 各変更単位 `CH-xxx` について、上表の特性を上から順に当て、影響しうる特性を選ぶ。
2. 影響しうる特性は、具体的なリスク（`RK-xxx`）と確認観点（`VP-xxx`）に落とす。
3. 影響しないと判断した特性は、分析レポートの非機能セクションに「対象外（根拠）」として簡潔に残す。
4. トリガー例はあくまで手掛かり。差分にトリガーがあっても影響が小さいと判断する場合は根拠を添える。

## 注意

- 機能テストで吸収できる項目と、非機能として別途確認すべき項目を区別する。
- 「性能が心配」で止めず、何を・どの条件で・どう確認するかまで書く（測定対象、負荷条件、判定の手掛かり）。
- 単一PRでは顕在化しにくい非機能リスク（移行順序、互換、スケール）はStep 4の横断サマリへ引き継ぐ。
