# テスト技法カタログ

## 入力値・境界系

| 技法 | 適用するリスク | 確認観点 |
|---|---|---|
| 同値分割 | 入力値、必須・任意、コード値、マスタ存在有無、有効値・無効値の判定誤り | 有効値、無効値、未入力、NULL、空白、存在するマスタ値、存在しないマスタ値 |
| 境界値分析 | 最大・最小桁数、金額、数量、比率、日付、期間、上限・下限、年度末、月末 | 最小値、最小値未満、最大値、最大値超過、0、1、-1、小数、月初、月末、期間内外 |

## 条件・状態系

| 技法 | 適用するリスク | 確認観点 |
|---|---|---|
| デシジョンテーブル | 権限、承認状態、締め状態、複数フラグ、業務区分、計算ロジック切替 | 条件組合せ、結果の一意性、矛盾、未定義組合せ、禁止操作、許可操作 |
| 状態遷移テスト | 申請、承認、差戻し、取消、締め、再実行、再承認、再締め | 正常遷移、禁止遷移、状態別操作可否、状態変更後の表示・出力・連携、異常終了時の中間状態 |
| 組合せテスト・ペアワイズ | 権限、状態、区分、期間、データ種別、環境差異など全組合せが現実的でない場合 | 重要条件、高リスク組合せ、ペアワイズで十分か、3因子以上が必要か、禁止条件の漏れ |

## 業務・経験則系

| 技法 | 適用するリスク | 確認観点 |
|---|---|---|
| シナリオテスト | 業務フロー全体、複数機能連鎖、後続影響、例外処理、再処理 | 実業務の流れ、前工程から後工程への反映、状態・データ一貫性、例外後の再開、帳票・連携結果 |
| エラー推測 | 過去障害、仕様の抜け、誤操作、開発者が見落としやすい条件、複雑な運用手順 | 類似障害、よくある入力ミス、例外動作、仕様の曖昧さ、実運用で起こる条件 |
| 回帰テスト | 共通処理変更、既存帳票、既存IF、既存データ互換、過去障害再発、対象外機能への副作用 | 変更対象外の主要機能、既存帳票差分、過去障害再発条件、既存データ、既存運用手順 |

## 出力・連携系

| 技法 | 適用するリスク | 確認観点 |
|---|---|---|
| 期待値検証・突合・差分確認 | 金額計算、集計、帳票、丸め、連携ファイル、明細・合計不一致、変更前後差分 | 明細と合計、集計条件、丸め・桁数・符号・単位、画面と帳票、出力ファイル、独立算出期待値 |
| インターフェーステスト | 外部システム、API、ファイル、バッチ、項目欠落、形式不一致、連携失敗、再送・再実行 | 正常連携、不正データ、必須欠落、形式不正、桁数超過、重複防止、エラー追跡、一部成功・失敗 |

## 非機能・統制系

| 技法 | 適用するリスク | 確認観点 |
|---|---|---|
| 性能・負荷・ボリュームテスト | 大量データ、同時利用、処理時間制約、バッチ、集計、月末・年度末ピーク | 想定データ量、同時利用、許容時間、タイムアウト、処理遅延時の判断、運用回避策 |
| ユーザビリティ・運用シナリオ確認 | メッセージ、複雑な手順、運用手順との不整合、異常時判断、手作業補正 | 次操作の分かりやすさ、原因と対処、業務実態との一致、誤操作防止、警告、手順書整合 |
| 互換性・構成テスト | OS、ブラウザ、Excel、PDF、画面解像度、文字コード、外部ツール差異 | サポート環境、主要環境優先、表示・出力・操作、外部アプリ連携、非サポート環境の扱い |
| 権限・セキュリティ観点確認 | 権限外参照・更新、承認者以外の承認、職務分離、監査証跡、URL直打ち、情報露出 | ロール別参照範囲、操作権限、直接アクセス拒否、承認者分離、監査ログ、機密情報露出 |
