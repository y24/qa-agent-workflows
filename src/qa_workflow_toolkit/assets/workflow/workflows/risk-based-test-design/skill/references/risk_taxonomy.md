# リスク分類リファレンス

## 目的

リスク候補を整理し、抽出漏れや偏りを防ぐための標準分類を定義する。

分類は整理の補助であり、分類そのものを目的にしない。1つのリスクには主分類を1つ選び、必要に応じて副分類を補足する。

## 標準分類

| 分類 | 概要 |
|---|---|
| 業務リスク | 業務処理、業務判断、業務結果に悪影響が出るリスク |
| 機能リスク | 入力、保存、更新、削除、検索、表示、計算、出力などが仕様どおりに動作しないリスク |
| データリスク | データの不整合、欠落、重複、履歴、桁数、単位、期間などに関するリスク |
| 連携リスク | 外部IF、API、バッチ、ファイル連携などの受け渡しに関するリスク |
| 権限・統制リスク | 権限、承認、職務分離、監査証跡、不正操作防止に関するリスク |
| 非機能リスク | 性能、可用性、操作性、互換性、セキュリティなどのリスク |
| 回帰リスク | 既存機能への副作用、既存データ影響、過去障害再発に関するリスク |
| 運用リスク | 運用手順、例外対応、リカバリ、締め日運用などに関するリスク |

## 主分類の選び方

| 判断観点 | 主分類候補 |
|---|---|
| 業務結果や業務判断への影響が中心 | 業務リスク |
| 単一機能の動作が中心 | 機能リスク |
| データ整合性、集計、履歴、桁数、丸めが中心 | データリスク |
| 外部システム、バッチ、APIとの受け渡しが中心 | 連携リスク |
| 権限、承認、監査、職務分離が中心 | 権限・統制リスク |
| 性能、可用性、操作性、互換性などが中心 | 非機能リスク |
| 変更による既存機能影響が中心 | 回帰リスク |
| 運用手順、例外対応、再実行、リカバリが中心 | 運用リスク |

複数にまたがる場合は、最終的に「どの悪影響を最も避けたいか」で主分類を決める。
