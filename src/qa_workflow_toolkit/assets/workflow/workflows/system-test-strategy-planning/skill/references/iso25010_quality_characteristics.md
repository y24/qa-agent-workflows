# ISO/IEC 25010 Quality Characteristics

## 目的

システムテスト観点を ISO/IEC 25010 のソフトウェア品質特性へ対応づけるための分類軸を定義する。

この分類は観点整理のために使う。入力根拠のない観点を追加するためには使わない。

## 品質特性

| 品質特性 | 主な意味 | システムテスト戦略での扱い |
|---|---|---|
| 機能適合性 | 明示・暗黙のニーズを満たす機能を提供できること | 要件、業務フロー、画面、API、帳票、バッチの確認観点に対応する |
| 性能効率性 | 実行時の性能、容量、資源使用が適切であること | 応答時間、処理時間、スループット、リソース、容量条件に根拠がある場合に扱う |
| 互換性 | 他システム、製品、環境と共存・相互運用できること | 外部IF、ブラウザ、OS、端末、ファイル形式、連携先に根拠がある場合に扱う |
| 使用性 | 利用者が有効、効率的、満足に利用できること | 操作性、エラー理解、入力支援、アクセシビリティ、利用者種別に根拠がある場合に扱う |
| 信頼性 | 必要な期間・条件で機能し続け、障害から回復できること | 可用性、障害時動作、再実行、データ保護、復旧に根拠がある場合に扱う |
| セキュリティ | 情報とデータを保護し、適切なアクセス制御を行えること | 認証、認可、監査、機密性、完全性、脆弱性対策に根拠がある場合に扱う |
| 保守性 | 修正、解析、変更、テストがしやすいこと | ログ、監視、設定、運用保守、変更影響、テスト容易性に根拠がある場合に扱う |
| 移植性 | 他環境へ移行・導入・適応しやすいこと | デプロイ、環境差分、移行、設定変更、基盤変更に根拠がある場合に扱う |

## マトリクス作成ルール

- 行は品質特性、列はテスト種別またはテスト観点のどちらかに統一する。
- 対応があるセルには Viewpoint ID を記載する。
- 対応がないセルは空欄ではなく、必要に応じて `根拠なし`、`対象外`、`要確認` を明記する。
- 1つの観点が複数特性に関係する場合、主特性と副特性を区別する。
- 機能適合性は非機能ではないが、システムテスト全体の品質特性マトリクスでは分類対象に含める。
