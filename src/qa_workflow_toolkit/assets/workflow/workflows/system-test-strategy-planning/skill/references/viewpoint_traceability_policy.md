# Viewpoint Traceability Policy

## 目的

テスト観点ごとに、根拠となった入力情報と後続テスト設計への引き継ぎを追跡できるようにする。

## 必須リンク

各 Viewpoint ID には、最低限次を紐づける。

- 関連 Test Type ID
- 関連 Coverage ID
- 主品質特性
- 根拠 Input ID または Source Fact ID
- 根拠要約
- 未確認事項
- 詳細テスト設計への引き継ぎ内容

## 推奨トレース

`Input ID -> Source Fact ID -> Coverage ID -> Test Type ID -> Viewpoint ID`

前段workflowを利用する場合は、前段IDも併記する。

## 根拠レベル

| 根拠レベル | 意味 | 扱い |
|---|---|---|
| 明示 | 入力文書に要求、条件、対象、制約が明記されている | 計画対象にできる |
| 関連明示 | 直接のテスト要求ではないが、対象や制約が明記されている | 観点化できるが条件不足は確認事項にする |
| 前段成果物 | 仕様抽出やリスク分析などの成果物IDが根拠 | 元文書参照が追える場合に利用する |
| 要確認 | 必要性は読み取れるが根拠や条件が不足 | 最終計画では未確定扱いにする |
| 根拠不足 | 入力から説明できない | 最終観点に採用しない |

## トレーサビリティ付属資料の列

- Viewpoint ID
- 観点名
- Test Type ID
- Coverage ID
- 主品質特性
- 副品質特性
- 根拠 Input ID
- 根拠 Source Fact ID
- 根拠要約
- 根拠レベル
- 未確認事項
- 詳細設計への引き継ぎ
