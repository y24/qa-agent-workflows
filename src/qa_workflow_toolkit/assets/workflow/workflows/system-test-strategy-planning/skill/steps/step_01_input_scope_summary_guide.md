# Step 1: 入力文書整理とシステムテスト対象スコープ確認

## 目的

入力文書と前段成果物を棚卸しし、システムテスト戦略の対象範囲、対象外、判断できない事項を整理する。

## 入力

- ユーザーが指定した開発文書、フォルダ、前段成果物
- `../../shared/input_document_handling.md`
- `references/system_test_scope_policy.md`
- `templates/input_scope_summary_template.md`

## 手順

1. 入力文書を `IN-001` 形式で一覧化する。
2. 文書ごとに種類、対象領域、版、参照可能な章・節・IDを記録する。
3. システムテスト戦略で扱える対象機能、非機能、外部IF、運用条件、環境制約を抽出する。
4. 明確な対象外、根拠不足、情報不足、矛盾を分けて記録する。
5. このStepではテスト種別やテスト観点を確定しない。

## 禁止事項

- 文書にない対象範囲を補完しない。
- 「一般に必要」だけで非機能やテスト種別を追加しない。
- 詳細テストケースを作らない。

## 出力

`01_input_scope_summary.md`

## 完了条件

- 入力文書と参照元が追跡できる。
- システムテスト戦略の対象、対象外、情報不足が分かれている。
- 次Stepでカバレッジ候補を抽出するための根拠が揃っている。
