# 根拠と確度ルール

## 目的

抽出、正規化、テスト設計インプット化において、参照元と確度を一貫して扱う。

## 根拠の残し方

各項目には可能な限り以下を残す。

- Doc ID
- 文書名
- 章、節、項目名
- ページ、表、画面、API名などの位置情報
- 原文要約
- 関連Raw ID / Spec ID / TDI ID

## 確度

| 確度 | 定義 | 扱い |
|---|---|---|
| 高 | 明確な記述があり、現行仕様として扱える | 通常の仕様として扱う |
| 中 | 根拠はあるが条件、範囲、版、優先順位に不明点がある | 注意付きで扱う |
| 低 | 根拠が弱い、古い、参考資料由来、推測に近い | 確認事項または仮扱い |

## 状態

| 状態 | 意味 |
|---|---|
| confirmed | 根拠が明確で矛盾がない |
| ambiguous | 曖昧で解釈が複数ありうる |
| conflicting | 文書間または同一文書内で矛盾がある |
| missing_detail | テスト設計に必要な粒度が不足している |
| tentative | 未確定、仮仕様、確認待ち |
| out_of_scope | 対象外 |
