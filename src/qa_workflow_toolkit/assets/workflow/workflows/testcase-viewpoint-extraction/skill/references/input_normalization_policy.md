# 入力正規化ポリシー

## 正規化項目

- TC ID
- 元IDまたは行番号
- テストケース名
- 前提条件
- 手順
- 期待結果
- 関連仕様・画面・機能
- 備考
- 入力状態

## issue化する入力問題

- 必須列がない
- 手順と期待結果が対応しない
- テストケース名が抽象的すぎる
- 前提条件が不足している
- 重複している
- 粒度が大きすぎる、または細かすぎる
- 元のIDが欠落している

## 大量テストケース

大量の場合はバッチに分割し、処理済み範囲と未処理範囲を明示する。分割してもID体系は一貫させる。

## 禁止事項

- 入力にない期待結果を補う
- 手順を独自に作り直す
- 不明点を黙って正規化する
