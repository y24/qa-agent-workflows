# 抽象化ポリシー

## 目的

既存テストケースの個別意図を、再利用可能な抽象観点へ変換する。

## 基本方針

- 根拠TCを必ず残す。
- 抽象化しすぎない。
- 観点名には条件と期待動作を残す。
- 個別名を残しすぎない。
- 根拠のない観点を追加しない。

## 抽象化レベル

| レベル | 内容 | 扱い |
|---|---|---|
| 低 | 個別テストケースに近い | カタログ化前に統合検討 |
| 中 | 条件と期待動作が残る再利用可能な観点 | 推奨 |
| 高 | 一般論に近く根拠TCとの対応が薄い | 抽象化しすぎとして見直す |

## 統合・分割

- 同じ条件、同じ期待動作、同じリスクを確認している候補は統合してよい。
- 条件、期待動作、対象状態、対象ロールが異なる場合は分ける。
- 複数観点を1つにまとめる場合は、根拠TCと含めた範囲を明記する。
