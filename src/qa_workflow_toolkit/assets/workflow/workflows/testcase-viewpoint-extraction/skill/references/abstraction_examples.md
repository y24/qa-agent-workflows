# 抽象化例

## 適切な抽象化

個別:

```text
管理者ユーザーで契約削除ボタンが表示されることを確認する
```

抽象観点:

```text
ロールごとに許可された操作のみが表示・実行できることを確認する
```

## 抽象化しすぎ

```text
権限を確認する
```

条件や期待動作が消えているため、根拠TCとの対応が弱い。

## 個別名を残しすぎ

```text
契約登録画面の契約削除ボタン表示確認
```

他機能へ再利用しにくい。対象が必要な場合は適用条件に移す。

## 分割すべき例

- 入力値の境界確認
- 権限別の操作可否確認
- 状態遷移後の表示確認

これらは確認条件と期待動作が異なるため、1つの抽象観点に詰め込みすぎない。
