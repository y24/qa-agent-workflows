# finalization_policy.md

# トレーサビリティ最終化ポリシー

## 最優先事項

`TC ID -> Intent ID -> Candidate ID -> Viewpoint ID` の対応を維持する。

## 除外・保留するもの

- 根拠TCがない抽象観点
- 推定度が低くレビュー未了の観点
- 候補から抽象観点への対応が不明なもの
- 対象外と判断したもの
- 重複統合されたが統合理由がないもの

## 修正してよいもの

- 表記ゆれ
- ID参照の軽微な誤り
- issueの重複統合
- カテゴリ保留の明示
- レビュー要否の整理

## issue統合

同じ原因の issue は統合してよい。ただし、元Issue IDと影響範囲を残す。issueを削除して見かけを整えない。

## 最終サマリ

- 対象範囲
- 抽出結果サマリ
- 主な観点カテゴリ
- 主要な抽象観点
- レビュー推奨事項
- 利用上の注意
- 次アクション候補
