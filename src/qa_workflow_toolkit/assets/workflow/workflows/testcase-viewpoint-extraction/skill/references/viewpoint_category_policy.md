# 観点カテゴリ選択ポリシー

## 選択基準

1. 何が主な確認対象かを見る。
2. 条件と結果のどちらが中心かを見る。
3. 出力物が中心なら帳票・出力を優先する。
4. 操作種別より、確認したい制約やリスクの種類を優先する。
5. 迷う場合は分類保留にし、issue化する。

## 迷いやすい例

| 迷い | 判断 |
|---|---|
| 入力値 / 境界値 | 境界付近の確認が中心なら境界値 |
| 状態 / 権限 | 状態で可否が変わるなら状態。ロールで可否が変わるなら権限 |
| 表示 / 帳票 | 画面表示なら表示・UI。ファイルや帳票なら帳票・出力 |
| 業務フロー / 状態 | 一連の流れが中心なら業務フロー。状態別動作が中心なら状態 |
| エラー / バリデーション | 入力拒否ならバリデーション。処理失敗後の扱いならエラー |

## 禁止事項

- カテゴリ名を無断で増やす
- 根拠TCにないカテゴリを付ける
- 分類を整えるために観点の意味を変える
- 分類保留を隠す
