# Common Contract

## Scope

この文書は、QA agent workflow skills に共通する行動契約を定義する。
各 skill は、skill 固有の `rules.md` や `references/` より前提としてこの契約を参照する。

## Core Principles

- 入力資料、前段成果物、ユーザー指定に存在する内容を事実として扱う。
- 入力から合理的に導けるが明記されていない内容は、推測または仮説として分ける。
- 判断に必要な情報が不足している場合は、確認事項または issue log に残す。
- 要件、仕様、業務ルール、リスク、テスト条件、期待結果を根拠なしに創作しない。
- QA成果物は、後続工程で再利用できる粒度、ID、根拠、状態を持たせる。
- QA成果物の出力先は、ユーザー指定を優先し、指定がない場合は `output_location_policy.md` に従う。

## Common Output Sections

成果物には、必要に応じて以下を含める。

- 概要
- 入力と参照元
- 対象範囲と対象外
- 主成果物
- 根拠または参照ID
- 未確認事項、矛盾、制約
- 次ステップへの引き継ぎ

## Separation Rules

以下を同じ欄に混ぜない。

- 事実
- 推測
- 前提
- 未確認事項
- 提案
- 対象外

推測が必要な場合は `推測:` と明記し、根拠または確認事項を添える。

## ID Handling

- 既存IDは可能な限り維持する。
- 削除、統合、分割、追加を行った場合は理由を記録する。
- 後続工程で参照されるIDを変更した場合は、影響範囲を明示する。
