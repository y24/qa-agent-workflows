# ドキュメント棚卸しポリシー

## 文書種別

| 種別 | 例 |
|---|---|
| requirements | 要件定義、業務要件 |
| design | 基本設計、詳細設計 |
| screen | 画面仕様、UI仕様 |
| api | API仕様、外部IF仕様 |
| data | データ定義、ER図、コード表 |
| operation | 運用手順、移行手順 |
| release | リリースノート、変更要求 |
| reference | 議事録、メモ、補足資料 |

## 棚卸し項目

- Doc ID
- 文書名
- 種別
- 版、日付
- 作成元
- 正本 / 参考資料 / 旧版 / 不明
- 主な内容
- 抽出対象範囲
- 除外範囲
- 注意点

## 優先度

ユーザー指定、最新版、正本、正式な仕様書を優先する。優先度が不明な場合は推測せず、確認事項にする。

## 未処理範囲

量が多い場合は、処理済み範囲と未処理範囲を明示する。未処理範囲は後続Stepで根拠不足として扱う。

## 禁止事項

- 文書を読まずに種別や内容を断定する
- 旧版や参考資料を現行仕様として扱う
- 未処理範囲を隠す
