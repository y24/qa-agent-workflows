# gap_review_policy.md

# ギャップレビュー方針

## ギャップ種別

| 種別 | 内容 |
|---|---|
| conflict | 文書間、同一文書内、前後Step間の矛盾 |
| missing | 仕様として必要だが記載がない |
| insufficient_detail | テスト設計に必要な条件・値・状態・期待結果が不足 |
| ambiguous | 複数解釈できる曖昧な記述 |
| tentative | 未確定、TODO、要検討、別途確認 |
| weak_evidence | 参考資料、旧版、根拠弱の情報 |
| unprocessed | 未処理範囲 |

## 優先度

| 優先度 | 目安 |
|---|---|
| High | 後続テスト設計で誤った判断につながる。中核機能、金額、権限、状態、連携に影響 |
| Medium | 一部のテスト観点やケース化に影響する |
| Low | 表記ゆれ、補足不足、後続で吸収可能 |

## 推奨対応

- ユーザー確認
- 文書追加確認
- 正本・最新版の確認
- 仕様決定待ち
- 後続テスト設計で仮扱い
- 対象外として明示

## レビュー観点

- Raw ID、Spec ID、TDI IDへの追跡が残っているか
- 矛盾や不足が成果物から消えていないか
- 後続テスト設計で断定禁止の事項が明示されているか
- 未処理範囲が明示されているか
- 確認先と影響が分かるか
