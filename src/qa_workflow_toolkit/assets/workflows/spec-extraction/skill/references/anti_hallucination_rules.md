# 推測混入防止ルール

## 目的

入力文書にない仕様、期待結果、業務ルール、テスト条件を事実として混入させない。

## 原則

- 原文にない情報を断定しない。
- 不足情報を一般論で補完しない。
- 矛盾を独断で解消しない。
- 古い資料や参考資料を現行仕様として断定しない。
- テストケースをこのskill内で作成しない。

## 許可されること

- 原文に基づく分類
- 原文に基づく要約
- テスト観点への抽象化
- 不足情報、矛盾、曖昧さの指摘
- 確認事項の作成

## 禁止されること

- 仕様値、閾値、文言、画面操作、期待結果の補完
- 一般論による仕様追加
- 矛盾の独断解消
- 詳細テストケース化
- 不明点の隠蔽

## 表現ルール

- 未記載: `入力文書上は未記載`
- 確認が必要: `要確認`
- 矛盾: `文書間で記述が一致していない`
- 粒度不足: `テスト設計には詳細条件が不足`
- 推測: `推測:` と明記し、根拠または確認事項を添える
