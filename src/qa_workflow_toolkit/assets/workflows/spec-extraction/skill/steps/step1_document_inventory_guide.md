# step1_document_inventory_guide.md

# Step 1: ドキュメント棚卸しガイド

## 目的

入力文書を棚卸しし、仕様抽出で使う対象範囲、文書種別、優先度、未処理範囲を整理する。

このStepでは仕様抽出、正規化、テスト設計インプット化を行わない。

## 入力

- ユーザーが指定した開発文書、要件、設計、画面、API、業務ルール、リリースノートなど
- 文書優先度、対象範囲、除外範囲の指定があればそれを含める

## 出力

- `document_inventory.md`
- 形式: `templates/document_inventory_template.md`

## 手順

1. 入力文書を Doc ID 付きで一覧化する。
2. 文書種別、版、日付、作成元、正本/参考資料の区別を整理する。
3. 抽出対象範囲、除外範囲、未処理範囲を記録する。
4. 文書間の優先順位や古い可能性がある資料を記録する。
5. Step 2で優先的に読む文書と注意点を引き継ぐ。

## 判断基準

- 文書種別、正本/参考資料、未処理範囲: `references/document_inventory_policy.md`
- 根拠と信頼度: `references/evidence_and_confidence_rules.md`

## このStepでやらないこと

- 仕様情報の抽出
- 矛盾の解消
- 不足仕様の補完
- テスト観点やテストケースの作成

## 完了条件

- 入力文書が Doc ID で管理されている
- 抽出対象、対象外、未処理範囲が明確
- Step 2で読むべき文書と注意点が整理されている

## 停止

`document_inventory.md` を出力したら停止する。ユーザーから明示的な指示があるまで Step 2 へ進まない。
