# 入力サマリ整理ポリシー

## 目的

Step 1で、後続の業務フロー整理とシナリオ設計に必要な情報を過不足なく圧縮する。

## 整理対象

- 対象システム、案件、対象業務、開発・改修背景
- テスト対象範囲、対象外、制約、前提条件
- 利用者、ロール、主要機能
- 画面、帳票、出力、外部連携、バッチ、後続処理
- 主要データ、マスタ、状態、ステータス
- 業務ルール、例外、エラー条件
- 過去不具合、既知リスク、確認事項

## 粒度

- 後続Stepで参照できるよう、機能名だけでなく業務上の意味を添える。
- 原文を長く転載せず、文書名、章、節、項目名で根拠を示す。
- 文書間の矛盾や未記載は補完せず、確認事項にする。
- Step 2でフロー化する必要がある領域を明示する。

## 対象外・制約

対象外や制約は、理由と後続Stepでの扱いを記録する。

例:

| 種別 | 書き方 |
|---|---|
| 対象外 | 今回リリース対象外のため、シナリオ候補化しない |
| 制約 | 外部接続不可のため、連携結果は代替確認が必要 |
| 未確定 | 仕様未確定のため、確認事項として次Stepへ引き継ぐ |

## 禁止事項

- 入力にない業務フローを確定事項として作る
- シナリオ観点やテストケースを先に作る
- 不明点を一般論で補完する
- 重要な対象外や制約を省略する
