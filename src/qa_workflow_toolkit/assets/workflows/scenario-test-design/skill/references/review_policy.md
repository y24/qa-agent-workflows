# レビュー・再実行ポリシー

## Step別レビュー観点

| Step | 主な確認 |
|---|---|
| 1 | 入力根拠、対象範囲、対象外、制約、確認事項が整理されているか |
| 2 | 業務目的を持つフローになっているか。画面遷移だけになっていないか |
| 3 | 観点がシナリオ化可能な粒度で、Flow IDや根拠と紐づいているか |
| 4 | シナリオ候補が観点をカバーし、優先度・採用判断・未カバーが明示されているか |
| 5 | テストケースがシナリオ、観点、フローへ追跡でき、未テストや対象外が隠れていないか |

## 共通レビュー観点

- 入力文書または前Step成果物に基づいているか
- 根拠のない断定がないか
- 推測、仮説、確認事項が事実と分離されているか
- 粒度が次Stepで扱いやすいか
- ID体系が一貫しているか
- 未確認事項が途中で消えていないか
- 次Stepへの引き継ぎが明確か

## 再実行ルール

- 既存IDは可能な限り維持する。
- 追加、削除、統合、分割した項目は理由を記録する。
- 後続成果物がある場合は影響範囲を明示し、必要な再実行Stepを提案する。
- High影響の指摘が残る場合は、次Stepへ進む前に修正する。
