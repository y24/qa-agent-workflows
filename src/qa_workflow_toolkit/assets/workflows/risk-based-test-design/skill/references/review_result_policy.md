# レビュー結果記録ポリシー

## 記録形式

```markdown
# レビュー結果

## 1. 総合判定

- 判定: OK / 条件付きOK / 要修正
- コメント:

## 2. 指摘一覧

| No | 対象成果物 | 指摘内容 | 重要度 | 対応方針 |
|---:|---|---|---|---|
| 1 |  |  | High / Medium / Low |  |

## 3. 修正が必要な事項

| No | 修正対象 | 修正内容 | 影響する後続Step |
|---:|---|---|---|
| 1 |  |  |  |

## 4. 未解決事項

| No | 内容 | 影響 | 対応方針 |
|---:|---|---|---|
| 1 |  |  |  |

## 5. 次Stepへ進む条件

- 
```

## 重要度

| 重要度 | 定義 | 対応 |
|---|---|---|
| High | 後続Stepの品質に大きく影響する。重要リスクの見落とし、誤評価、追跡不能など | 次Stepに進む前に修正する |
| Medium | 一部の精度や分かりやすさに影響する。粒度不揃い、根拠不足など | 可能なら次Step前に修正する |
| Low | 表記ゆれ、軽微な補足不足など | 後続Stepで吸収してよい |

## 次Stepへ進んでよい条件

- High指摘が残っていない
- 対象範囲が明確である
- 未確認事項が記録されている
- 後続Stepで扱うべき入力が揃っている
- ユーザーが明示的に続行を指示している

MediumまたはLowの指摘が残っている場合でも、ユーザーが許容すれば次Stepへ進んでよい。ただし、残っている指摘は後続Stepへの引き継ぎ事項として記録する。
