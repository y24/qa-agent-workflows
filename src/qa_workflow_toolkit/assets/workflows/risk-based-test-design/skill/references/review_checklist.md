# レビューチェックリスト

## 目的

各Stepの出力後、次Stepへ進む前にレビューする。入力文書にない内容の混入、推測と事実の混同、粒度不揃い、重要リスクの見落とし、評価の恣意性、トレーサビリティ切れを防ぐ。

## 共通レビュー観点

| No | チェック項目 | 判定 |
|---:|---|---|
| 1 | 入力文書または前Step成果物に基づいているか | OK / NG / 要確認 |
| 2 | 根拠のない断定が含まれていないか | OK / NG / 要確認 |
| 3 | 推測・仮説・確認事項が事実と分離されているか | OK / NG / 要確認 |
| 4 | 一般論だけで水増しされていないか | OK / NG / 要確認 |
| 5 | 出力の粒度が次Stepで扱いやすいか | OK / NG / 要確認 |
| 6 | 重要な対象範囲・制約・未確定事項が抜けていないか | OK / NG / 要確認 |
| 7 | 同じ内容が複数箇所で重複していないか | OK / NG / 要確認 |
| 8 | ユーザー確認が必要な事項が明示されているか | OK / NG / 要確認 |
| 9 | 次Stepに引き継ぐべき事項が明確か | OK / NG / 要確認 |

## 最終レビュー観点

| No | チェック項目 | 判定 |
|---:|---|---|
| 1 | Step 1の対象範囲が後続Stepに反映されているか | OK / NG / 要確認 |
| 2 | Step 2のリスクがStep 3で評価されているか | OK / NG / 要確認 |
| 3 | Step 3の優先度がStep 4のテスト深度に反映されているか | OK / NG / 要確認 |
| 4 | Step 4のテスト観点がStep 5のテストケース骨子に反映されているか | OK / NG / 要確認 |
| 5 | リスク、観点、テストケースのID体系が一貫しているか | OK / NG / 要確認 |
| 6 | 未確認事項が途中で消えていないか | OK / NG / 要確認 |
| 7 | 対象外、保留、残存リスクの扱いが一貫しているか | OK / NG / 要確認 |
| 8 | 高リスク領域にテストが重点配分されているか | OK / NG / 要確認 |
| 9 | 低リスク領域を過剰に詳細化していないか | OK / NG / 要確認 |
| 10 | 詳細テスト設計へ進める状態になっているか | OK / NG / 要確認 |
