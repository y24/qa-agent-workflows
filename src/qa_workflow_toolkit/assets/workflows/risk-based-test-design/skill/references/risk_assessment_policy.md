# リスク評価判断ポリシー

## 優先度ごとの扱い

| 優先度 | 扱い |
|---|---|
| P0 | 必ずテスト方針に含める。正常系、異常系、境界、状態遷移、組合せ、回帰影響を厚く確認する。未テストなら残存リスクを明示する。 |
| P1 | 原則としてテスト対象に含める。主要パターン、代表的な異常系、境界条件を確認する。 |
| P2 | 標準的な確認対象。代表値、主要条件、基本的な異常系を確認し、高優先ケースへ統合してよい。 |
| P3 | 簡易確認、探索的確認、対象外候補。他テストで同時確認できる場合は統合し、完全に消さず扱いを記録する。 |

## 補正ルール

- 影響度が5のリスクは、スコアが低くても原則 P1 以上を検討する。
- 会計、金額、請求、支払、契約、税務、法令、監査、内部統制、個人情報、権限・承認に関係する場合は、優先度を1段階上げることを検討する。
- 過去障害に類似するリスクは、発生可能性または検出困難度を高めに評価する。
- 共通部品、共通ロジック、共通マスタ、共通帳票、共通IFに関わる場合は、影響度または発生可能性を高めに評価する。
- 複数条件、特殊状態、大量データ、外部連携、権限組合せ、長い業務フローが必要な場合は、検出困難度を高めに評価する。
- 根拠信頼度が低くても影響が大きいリスクは削除しない。仮説リスク、確認事項、暫定優先度として残す。

## 評価理由

評価理由には、以下を含める。

- なぜその影響度にしたか
- なぜその発生可能性にしたか
- なぜその検出困難度にしたか
- 補正した場合は、なぜ補正したか
- 根拠信頼度が低い場合は、どこが不確実か

悪い例:

```text
重要そうなのでP1。
```

良い例:

```text
締め後データの不整合は、確定済み帳票と後続会計処理に影響するため影響度4。
締め状態、修正操作、帳票出力条件の組合せで発生しうるため発生可能性3。
通常の正常系確認では見逃しやすいため検出困難度4。
Scoreは48であり、P1とする。
```

## 評価保留

| 状態 | 扱い |
|---|---|
| 影響範囲が不明 | 影響度を暫定評価し、確認事項を作る |
| 発生条件が不明 | 発生可能性を暫定評価し、確認事項を作る |
| 検出方法が不明 | 検出困難度を高めに置くか、評価保留にする |
| 根拠が弱い | 根拠信頼度を低にする |
| 仕様未確定 | 暫定優先度を付け、保留事項にする |

## 偏り防止チェック

- すべてのリスクが P0/P1 または P2/P3 に偏っていないか
- 影響度と発生可能性を混同していないか
- 複雑さだけで影響度を高くしていないか
- 業務影響が大きいリスクを低く評価していないか
- 検出困難度や根拠信頼度を無視していないか
- 補正理由が書かれているか
- P0/P1の量がテスト制約上扱える範囲か
