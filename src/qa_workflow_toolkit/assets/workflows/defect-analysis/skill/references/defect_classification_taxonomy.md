# 不具合分類タクソノミー

## 使い方

- Step 2では仮分類の候補として使う。
- Step 3では分類軸として使う。
- チケット記述から判断できない場合は `分類保留` とする。
- 分類軸は分析目的に応じて増減してよいが、追加した分類は定義を明示する。

## 機能領域分類

| 分類 | 定義 |
|---|---|
| 画面 | 画面表示、画面操作、画面遷移に関する不具合 |
| 帳票 | 帳票出力、レイアウト、印字内容、出力条件に関する不具合 |
| 入力チェック | 入力値、必須、形式、範囲、相関チェックに関する不具合 |
| 計算処理 | 金額、数量、期間、丸め、集計など計算結果に関する不具合 |
| データ連携 | システム間連携、ファイル連携、API連携に関する不具合 |
| 権限 | ロール、認可、参照範囲、操作権限に関する不具合 |
| マスタ | マスタ参照、更新、組み合わせ、世代管理に関する不具合 |
| バッチ | 定期処理、非同期処理、リトライ、締め処理に関する不具合 |
| 外部IF | 外部サービス、外部システム、IF仕様差分に関する不具合 |
| インストール・更新 | インストール、アップデート、移行、配布に関する不具合 |
| 性能・タイムアウト | 応答時間、処理時間、タイムアウト、負荷に関する不具合 |
| 分類保留 | 情報不足または複数候補で判断できないもの |

## 不具合タイプ分類

| 分類 | 定義 |
|---|---|
| 仕様理解漏れ | 仕様記述の読み違い、前提理解不足、仕様反映漏れ |
| 条件分岐漏れ | 条件の一部、例外条件、組み合わせ条件の処理漏れ |
| 境界値・特殊値漏れ | 境界値、ゼロ、空、最大、最小、特殊文字などの扱い漏れ |
| データ不整合 | 保存、更新、参照、同期、整合性維持の不備 |
| 状態遷移不備 | ステータス、ライフサイクル、遷移条件の不備 |
| 入力チェック不備 | 入力制約、相関チェック、エラー表示の不備 |
| 表示・文言不備 | 表示内容、文言、レイアウト、フォーマットの不備 |
| 計算ロジック誤り | 計算式、丸め、集計、期間計算などの誤り |
| 権限制御不備 | 認可、参照制限、操作制限の不備 |
| エラーハンドリング不備 | 例外、失敗時処理、リトライ、メッセージの不備 |
| 環境依存 | OS、ブラウザ、設定、外部環境差分に依存する不具合 |
| 回帰不具合 | 過去正常だった機能が変更により不正になったもの |
| テストデータ・設定不備 | テストデータ、設定、環境準備に起因するもの |
| 分類保留 | 情報不足または複数候補で判断できないもの |

## 発見タイミング分類

| 分類 | 定義 |
|---|---|
| 単体テスト | モジュール、クラス、関数単位のテストで発見 |
| 結合テスト | コンポーネント間、機能間、IF間のテストで発見 |
| システムテスト | システム全体またはE2E相当のテストで発見 |
| 受入テスト | ユーザー受入、業務部門確認で発見 |
| リリース後 | 本番リリース後、利用開始後に発見 |
| 運用中 | 運用作業、監視、問い合わせから発見 |
| 不明 | チケットから判断できないもの |

## 影響分類

| 分類 | 定義 |
|---|---|
| 業務停止 | 主要業務が継続できない |
| データ破壊・不整合 | データの消失、破壊、不整合が発生する |
| 誤計算 | 金額、数量、期間などの計算結果が誤る |
| 誤表示 | 表示内容が事実や仕様と異なる |
| 操作不能 | 特定操作、画面、処理が実行できない |
| 操作性低下 | 業務継続は可能だが効率や理解性が低下する |
| 軽微な表示問題 | 業務影響が小さい文言、余白、軽微な見た目の問題 |
| 不明 | 影響範囲が読み取れないもの |

## テスト観点との関係分類

| 分類 | 定義 |
|---|---|
| 既存テストで検出可能だった | 既存テスト条件で本来検出できた根拠がある |
| 既存テストでは検出困難だった | 環境、データ、運用条件などから検出困難だった根拠がある |
| テスト条件不足 | 条件、分岐、組み合わせ、状態の不足 |
| テストデータ不足 | データ種類、量、組み合わせ、履歴、異常データの不足 |
| 環境差分不足 | OS、ブラウザ、外部IF、設定、性能条件の不足 |
| 業務シナリオ不足 | 実業務の流れ、例外運用、長期利用シナリオの不足 |
| 回帰テスト不足 | 変更影響や既存機能確認の不足 |
| 判断不能 | 根拠不足で判断できないもの |
