# Run Step Prompt Template

```markdown
$defect-analysis の Step <番号> を実行してください。

## 対象

- 対象プロダクト・機能:
- 対象期間:
- 対象チケット:
- 分析目的:

## 入力

- <不具合チケット一覧または前Step成果物>
- <チケット項目定義>
- <関連するテスト設計・仕様・補足資料>

## 出力先

- <指定がなければ shared/output_location_policy.md に従う>

## 注意事項

- チケットにない原因、仕様、期待結果、再発防止策を創作しない。
- 事実、推測、確認事項を分ける。
- 根拠チケットIDまたは前Step成果物IDを残す。
- Step完了後は次Stepへ進まず、レビュー待ちで停止する。
```
