# 曖昧さ・推定・issue管理ポリシー

## 目的

既存テストケースから意図や観点を読む際の曖昧さ、推定度、レビュー要否、issue記録を管理する。

## 推定度

| 推定度 | 定義 | 扱い |
|---|---|---|
| 高 | 手順・期待結果から意図が明確 | 通常採用 |
| 中 | 意図は読めるが条件や期待動作に不明点がある | 注意付き採用、必要に応じてレビュー |
| 低 | 複数解釈があり根拠が弱い | issue化し、確定扱いしない |

## レビュー要否

レビュー要にする例:

- 意図が複数に読める
- 期待結果が曖昧
- 前提条件が不足
- 手順と期待結果が対応していない
- 根拠TCが少ない
- 抽象化レベルに迷う

## issue 種別

| 種別 | 内容 |
|---|---|
| input_gap | 入力欠落、列不足、情報不足 |
| ambiguity | 複数解釈、曖昧な期待結果 |
| weak_evidence | 根拠TCが弱い、少ない |
| conflict | 手順と期待結果、TC間の矛盾 |
| abstraction | 抽象化しすぎ、個別すぎ、統合迷い |
| traceability | 対応関係が切れている |
| out_of_scope | 対象外、扱わない項目 |
