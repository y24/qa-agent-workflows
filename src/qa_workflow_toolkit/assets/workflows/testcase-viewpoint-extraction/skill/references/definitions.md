# 用語定義

## 目的

既存テストケースから意図、観点候補、抽象観点、トレーサビリティを抽出する際の用語を定義する。

## 主要用語

| 用語 | 定義 |
|---|---|
| テストケース | 入力、前提、手順、期待結果などを持つ既存の確認単位 |
| テスト意図 | そのテストケースが何を確認しようとしているか |
| 主意図 | テストケースの中心目的 |
| 副次意図 | 主目的に付随して確認している内容 |
| 観点候補 | 意図から抽出した、抽象観点化前の候補 |
| 抽象観点 | 複数テストケースに再利用できる確認観点 |
| 観点カテゴリ | 抽象観点を整理する分類 |
| 根拠TC | 観点や意図の根拠となるテストケース |
| 推定度 | 意図や観点をどの程度確からしく読めるか |
| レビュー要否 | 人間レビューが必要かどうか |
| issue | 曖昧さ、根拠不足、未対応、矛盾、保留などの記録 |
| トレーサビリティ | TC、意図、候補、抽象観点の対応関係 |

## ID体系

| 対象 | 形式 |
|---|---|
| Test Case | `TC-001` |
| Intent | `INT-001` |
| Candidate | `VC-001` |
| Viewpoint | `VP-001` |
| Issue | `ISS-001` |
