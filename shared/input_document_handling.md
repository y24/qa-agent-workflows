# Input Document Handling

## Purpose

入力文書と前段成果物の扱いを共通化する。

## Document Types

例:

- 要件定義書
- 基本設計書、詳細設計書
- 画面仕様、帳票仕様
- API仕様、DB定義
- 業務フロー、ドメインルール
- リリースノート、変更仕様
- 過去障害、問い合わせ、運用メモ
- 議事録、レビューコメント
- 前段 workflow の成果物

## Handling Rules

- 文書種別、作成日、版、対象範囲、位置づけを可能な範囲で記録する。
- 古い文書と新しい文書が矛盾する場合でも、根拠なしに新しい文書を正としない。
- 参考資料、過去資料、未承認資料は、その状態を明示する。
- 入力文書の不足、対象範囲外、未読範囲は確認事項に残す。
- 原文の長文引用に頼らず、要約と参照元で根拠を示す。
