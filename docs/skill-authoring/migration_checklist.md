# Migration Checklist

既存プロンプト群や大型 `SKILL.md` を、標準的な skill 構成へ再編成する際に使う。
スクリプトで機械的に分割するのではなく、各ファイルの責務を確認して移す。

## 1. 対象確認

- [ ] 対象 skill 名とディレクトリを確認した。
- [ ] 現在の `SKILL.md`、`steps/`、`references/`、`templates/`、`examples/` を棚卸しした。
- [ ] 既存のステップ ID、成果物 ID、参照 ID を確認した。
- [ ] 通常実行に必要な内容と、保守者向け説明を分けた。
- [ ] 複数 skill で共通化できる内容を確認した。

## 2. 内容の分類

- [ ] 目的、起動条件、入出力、workflow 概要を `SKILL.md` に分類した。
- [ ] 実行順序、レビュー待ち、引き継ぎを `orchestrator.md` に分類した。
- [ ] skill 固有の禁止事項、品質基準、判断ルールを `rules.md` に分類した。
- [ ] 個別作業の手順を `steps/` に分類した。
- [ ] 判断基準、分類、観点、用語を `references/` に分類した。
- [ ] 成果物形式を `templates/` に分類した。
- [ ] 具体例や完成例を `examples/` に分類した。
- [ ] 複数 skill の共通ルールを `shared/` に分類した。

## 3. 分割と配置

- [ ] ファイル名は目的が分かる名前にした。
- [ ] `part1`、`part2`、`misc` のような名前を使っていない。
- [ ] `SKILL.md` は入口として必要最小限にした。
- [ ] 各 `steps/*.md` は1ステップ1責務にした。
- [ ] 長い判断基準は `references/` に移した。
- [ ] 長いテンプレートは `templates/` に移した。
- [ ] 通常実行で毎回不要な例は `examples/` に移した。
- [ ] 100行を超える `references/*.md` には目次を置いた。

## 4. 整合性確認

- [ ] `SKILL.md` から参照するファイルが存在する。
- [ ] 移動前後で主要な業務内容が欠落していない。
- [ ] 同じルールの重複記載が増えていない。
- [ ] 前提、推測、未確認事項の扱いが維持されている。
- [ ] トレーサビリティの記載方針が維持されている。
- [ ] レビューゲートの位置が維持されている。
- [ ] 出力テンプレートの列名と章立てが安定している。

## 5. サイズ確認

- [ ] `SKILL.md` が300行を超えていない。
- [ ] `steps/*.md` が250行を超えていない。
- [ ] `references/*.md` が300行を超えていない。
- [ ] `templates/*.md` はできるだけ100行以内に収まっている。
- [ ] 行数内でも、責務が混在するファイルを残していない。

## 6. 最終確認

- [ ] `rg` などで古いファイル名への参照が残っていないことを確認した。
- [ ] 追加・変更したファイルの差分を確認した。
- [ ] 影響範囲を説明できる状態にした。
- [ ] 未解決の前提や確認事項を記録した。
