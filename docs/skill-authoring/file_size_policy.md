# File Size Policy

この文書は、skill 作成・再構成時のファイルサイズ目安を定義する。
目的は、1回の作業で必要な範囲を読み切れる状態を保ち、通常実行時の文脈量を増やしすぎないことである。

## 行数の目安

| 種別 | 推奨上限 | 見直し目安 | 分割または再構成の目安 |
|---|---:|---:|---:|
| `AGENTS.md` | 120行 | 150行 | 200行超 |
| `SKILL.md` | 200行 | 250行 | 300行超 |
| `orchestrator.md` | 180行 | 220行 | 250行超 |
| `rules.md` | 180行 | 220行 | 250行超 |
| `steps/*.md` | 180行 | 220行 | 250行超 |
| `references/*.md` | 100行 | 200行 | 300行超 |
| `templates/*.md` | 100行 | 150行 | 200行超 |
| `examples/*.md` | 1テーマ | 150行 | テーマ混在時 |

`references/*.md` が100行を超える場合は、冒頭に目次を置く。
行数内であっても、複数の責務が混在している場合は分割する。

## 分割の判断基準

以下のどれかに該当する場合は、別ファイル化を検討する。

- 入口説明、実行手順、判断基準、テンプレート、例が同じファイルに混在している。
- あるステップの実行時に読まない情報が多い。
- 同じルールが `AGENTS.md`、`SKILL.md`、`steps/` に重複している。
- 目次なしでは目的の箇所を探しにくい。
- 1つの変更が、無関係な業務説明まで巻き込む。
- 将来の skill 追加時にコピーされやすい長文が含まれている。

## 分割先の選び方

| 内容 | 分割先 |
|---|---|
| 常時適用する最小限のリポジトリルール | `AGENTS.md` |
| skill 作成者向けの方針 | `docs/skill-authoring/` |
| 複数 skill が実行時に使う共通ルール | `shared/` |
| skill の目的、起動条件、入出力 | `SKILL.md` |
| workflow 全体の制御 | `orchestrator.md` |
| skill 固有の品質基準や禁止事項 | `rules.md` |
| 個別ステップの実行手順 | `steps/` |
| 判断基準、分類、カタログ、用語 | `references/` |
| 成果物フォーマット | `templates/` |
| 具体例、完成例、良い例・悪い例 | `examples/` |

## 避ける分割

- `part1.md`、`part2.md` のように目的が分からない分割。
- 行数だけを理由に、1つの判断基準を不自然に分断する分割。
- 同じ表やルールを複数ファイルに複製する分割。
- `SKILL.md` から大量の詳細ファイルを常時読む前提の分割。

## レビュー時の確認

- `SKILL.md` は入口として短く保たれているか。
- 各 `steps/*.md` は、そのステップで必要な手順だけを含んでいるか。
- 判断基準やカタログは `references/` に集約されているか。
- テンプレートは `templates/` に分離され、列名や章立てが安定しているか。
- 共通ルールが `shared/` に移され、重複が増えていないか。
- 既存の ID、参照、トレーサビリティが維持されているか。
