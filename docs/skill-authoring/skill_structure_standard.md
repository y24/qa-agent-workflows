# Skill Structure Standard

この文書は、`.agents/skills/<skill-name>/` 配下の標準構成を定義する。
既存 skill の再構成でも、新規 skill の作成でも、この構成を基準にする。

## 必須ファイル

| ファイル | 役割 |
|---|---|
| `SKILL.md` | skill の入口。目的、起動条件、入出力、workflow 概要、参照ファイル一覧を置く。 |
| `orchestrator.md` | 複数ステップの進行、レビュー待ち、成果物の引き継ぎを制御する。 |
| `rules.md` | skill 固有の禁止事項、品質基準、判断ルールをまとめる。 |

## 推奨ディレクトリ

| ディレクトリ | 役割 |
|---|---|
| `steps/` | 主要ステップごとの実行手順。1ファイル1ステップを基本にする。 |
| `references/` | 判断基準、分類、用語、観点カタログ、評価基準など。 |
| `templates/` | 成果物フォーマット。表の列名、章立て、出力順を安定させる。 |
| `examples/` | 通常実行時に必須ではない例、完成例、良い例・悪い例。 |

## `SKILL.md` に置く内容

- skill の目的。
- 使うべき状況と使わない状況。
- 主な入力。
- 主な成果物。
- workflow の概要。
- 実行時に参照するファイル一覧。
- この skill 固有の最小限の注意事項。

`SKILL.md` には、詳細な作業手順、長い判断基準、テンプレート本文、大量の例を置かない。

## `orchestrator.md` に置く内容

- ステップの実行順序。
- どの成果物を次ステップに渡すか。
- どこでユーザー確認を待つか。
- 入力不足、矛盾、対象外がある場合の分岐。
- 最終成果物のまとめ方。

## `rules.md` に置く内容

- skill 固有の品質基準。
- 禁止事項。
- 推測、前提、未確認事項の扱い。
- トレーサビリティの維持方法。
- 粒度や優先度の判断方針。

複数 skill で共通するルールは `shared/` に置き、`rules.md` では参照だけにする。

## `steps/` に置く内容

各ファイルは、1つの作業ステップを完了するために必要な手順に絞る。
判断基準、分類表、詳細テンプレート、長い例は `references/` または `templates/` に逃がす。

推奨構成:

```text
steps/
  step_01_<purpose>_guide.md
  step_02_<purpose>_guide.md
  step_03_<purpose>_guide.md
```

## `references/` に置く内容

- 分類基準。
- 評価基準。
- 用語定義。
- 観点カタログ。
- 設計原則。
- レビュー観点。

100行を超える場合は冒頭に目次を置く。300行を超える場合は、テーマ別に分割する。

## `templates/` に置く内容

- 成果物の Markdown 構成。
- 表の列名。
- 出力順。
- 記載粒度の短いガイド。

複数の大きな成果物テンプレートを1ファイルに詰め込まない。

## `examples/` に置く内容

1ファイル1テーマを基本にする。
長い完成例、良い例、悪い例は別ファイルに分ける。
通常の workflow が例を読まなくても実行できる状態にする。
