# リスクベースドテスト設計 実行用プロンプトテンプレート

## 1. 目的

このテンプレートは、リスクベースドテスト設計プロンプト群の各StepをAIエージェントに実行させるために使用する。

実行時は、対象Step、参照ファイル、入力文書、出力成果物を明示する。  
AIエージェントは、指定されたStepのみを実行し、成果物を出力したら停止する。

---

## 2. 基本実行ルール

AIエージェントは、以下のルールを守ること。

- 指定されたStepのみ実行する
- 次Stepへ勝手に進まない
- 指定された参照ファイルのみを優先的に使用する
- 入力文書にない内容を事実として扱わない
- 推測した内容は、仮説・確認事項・未確定事項として分離する
- 出力はMarkdown形式にする
- 成果物は、指定されたテンプレートに沿って作成する
- 不明点があっても作業を完全停止せず、確認事項として記録する
- 詳細化しすぎず、対象Stepの粒度を守る

---

## 3. 実行プロンプト雛形

以下をコピーし、必要箇所を埋めて使用する。

```markdown
# リスクベースドテスト設計 Step実行依頼

## 実行対象Step

- Step: <Step番号>
- Step名: <Step名>

## 参照する共通ルール

以下のファイルを参照してください。

- `rules.md`
- `orchestrator.md`

## 参照するStep Guide

以下のStep Guideに従ってください。

- `<step guide file name>`

## 参照する補助ファイル

必要に応じて、以下の補助ファイルを参照してください。

- `<reference file name>`

## 入力ファイル

このStepでは、以下を入力として扱ってください。

- `<input file 1>`
- `<input file 2>`
- `<input file 3>`

## 使用するテンプレート

成果物は、以下のテンプレートに沿って作成してください。

- `<template file name>`

## 出力成果物

以下の成果物をMarkdown形式で出力してください。

- `<output file name>`

## 実行時の注意

- このStepで扱う範囲に限定してください
- 次Stepの内容に踏み込みすぎないでください
- 根拠のない断定をしないでください
- 推測は仮説または確認事項として分離してください
- 不明点は確認事項として記録してください
- 成果物を出力したら停止してください
````

## 8. 最終整合性確認プロンプト例

Step 1〜Step 5完了後に、成果物全体の整合性を確認する場合は、以下を使用する。

## 確認観点

- Step 1の対象範囲が後続Stepに反映されているか

- Step 4のテスト観点がStep 5のテストケース骨子に反映されているか
- Risk ID、Viewpoint ID、TC IDの対応が追跡可能か
- P0/P1リスクに未カバーがないか

## 9. 運用上の推奨

このテンプレートを使用するときは、以下の運用を推奨する。

- Stepごとに成果物をレビューしてから次へ進む
- 各Stepで参照するファイルを最小限にする
- 前Step成果物を無制限に全文再掲しない
- 長大な入力文書は、Step 1で必ず要約・圧縮する
- 不明点は作業停止ではなく、確認事項として管理する
- P0/P1リスクは人間が重点レビューする
- 最終成果物は、詳細テストケース作成前に整合性確認する

## 10. 実行時に埋める項目一覧

| 項目 | 内容 |
|---|---|
| Step番号 | 実行するStepの番号 |
| Step名 | 実行するStepの名称 |
| Step Guide | 対象Stepのguideファイル |
| 補助ファイル | risk_taxonomy、scoring_modelなど |
| 入力ファイル | 前Step成果物または開発ドキュメント |
| テンプレート | 対象Stepの成果物テンプレート |
| 出力成果物 | 作成するMarkdownファイル名 |
| 注意事項 | 今回のプロジェクト固有の制約や補足 |
