# テスト技法マッピングリファレンス

## 目的

Step 4で、リスクの性質に応じたテスト観点、確認条件、推奨技法を選ぶ。

技法名を付けること自体を目的にしない。リスクを検出しやすい確認条件を作り、高リスク領域にテストを集中させる。

## リスクの性質と推奨技法

| リスクの性質 | 主な確認対象 | 推奨技法 |
|---|---|---|
| 入力値、金額、数量、日付、期間 | 入力範囲、桁数、境界、単位 | 同値分割、境界値分析 |
| 条件分岐、判定、権限、承認可否 | 条件の組合せ、可否判断 | デシジョンテーブル |
| 状態、ステータス、締め、取消、再実行 | 状態遷移、遷移可否、状態別動作 | 状態遷移テスト |
| 業務フロー、複数機能の連鎖 | 一連の業務処理、後続影響 | シナリオテスト |
| 複数条件の組合せ | 条件組合せ、パターン圧縮 | 組合せテスト、ペアワイズ |
| 過去障害、経験則上の弱点 | 再発確認、抜け漏れ確認 | エラー推測、回帰テスト |
| 帳票、集計、計算 | 期待値、集計結果、差分 | 期待値検証、突合、差分確認 |
| 外部IF、バッチ、API | 連携成功、連携失敗、再実行 | インターフェーステスト、異常系テスト、再実行テスト |
| 性能、処理時間、大量データ | 応答時間、処理時間、スループット | 性能テスト、負荷テスト、ボリュームテスト |
| 操作性、メッセージ、運用 | 利用者判断、復旧容易性 | ユーザビリティ確認、運用シナリオ確認 |
| 互換性、環境差異 | OS、ブラウザ、Excel、PDF、端末差異 | 互換性テスト、構成テスト |
| セキュリティ、権限、情報保護 | 不正アクセス、情報露出、監査証跡 | 権限テスト、セキュリティ観点確認 |

## リスク分類別の主な技法

| リスク分類 | 主に使う技法 |
|---|---|
| 業務リスク | シナリオテスト、状態遷移テスト、デシジョンテーブル |
| 機能リスク | 同値分割、境界値分析、デシジョンテーブル |
| データリスク | 境界値分析、期待値検証、突合、差分確認 |
| 連携リスク | インターフェーステスト、異常系テスト、再実行テスト |
| 権限・統制リスク | デシジョンテーブル、権限テスト、状態遷移テスト |
| 非機能リスク | 性能テスト、互換性テスト、ユーザビリティ確認 |
| 回帰リスク | 回帰テスト、差分確認、エラー推測 |
| 運用リスク | 運用シナリオ確認、異常系テスト、再実行テスト |

## 優先度別の技法選定

| 優先度 | 方針 |
|---|---|
| P0 | 単一技法に頼らず、シナリオ、状態遷移、条件組合せ、境界、回帰、異常系、期待値検証を必要に応じて組み合わせる |
| P1 | 主要な正常系、代表的な異常系、重要な境界条件を確認する |
| P2 | 代表条件を中心に確認し、P0/P1のテストケースへ統合してよい |
| P3 | 簡易確認、他テストへの統合、探索的確認、レビュー確認、対象外を検討する |
