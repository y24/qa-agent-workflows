# test_design_input_catalog_template.md

## 1. このテンプレートの目的

このテンプレートは、Step 4「テスト設計インプット化」の成果物である `test_design_input_catalog.md` を作成するためのものである。

`test_design_input_catalog.md` の目的は、`normalized_spec_inventory.md` に整理された仕様情報を、後続のテスト設計工程で利用できる形に変換することである。

このテンプレートでは、以下を行う。

- テスト対象を整理する
- 確認すべき仕様を整理する
- テスト観点を抽出する
- 必要条件・前提を整理する
- 入力データ観点を整理する
- 期待結果の種類を整理する
- 関連コンポーネントを整理する
- 優先度とリスクを付与する
- 参照元となる `SPEC-ID` を記録する

このテンプレートでは、以下を行わない。

- 詳細なテストケースを作成する
- テスト手順を作成する
- 具体的なテストデータを網羅展開する
- 期待結果を推測で断定する
- 未確定仕様を確定扱いする

---

## 2. 使用ルール

このテンプレートを使用する際は、以下を守る。

- 各テスト設計インプットに `TDI-ID` を付与する
- 各項目に参照元 `SPEC-ID` を記録する
- 仕様状態が未確定・矛盾・不足の場合は、断定表現を避ける
- 期待結果は「種類」までを基本とする
- 具体的な期待結果は、元仕様に明記されている場合のみ記載する
- 入力データは「観点」までに留める
- テストケースID、操作手順、具体的な検証手順は作成しない
- 後続のテスト設計で確認が必要な事項を明示する
