# 確認方法カタログ

## 確認方法分類

| 分類 | 用途 | 例 |
|---|---|---|
| テストで確認 | 実際に動かして測定・観察する | 性能テスト、負荷テスト、互換性テスト、権限テスト、障害復旧テスト |
| 機能テストに組み込む | 専用非機能テストにせず通常テスト内で確認する | エラー時の入力保持、権限別表示、画面遷移後の文脈維持 |
| レビューで確認 | 設計、構成、手順、コード、設定の確認が適する | ログ設計、監査証跡、例外処理方針、保守性、バージョン方針 |
| 静的解析・ツールで確認 | ツール判定や定点観測が適する | 脆弱性スキャン、依存ライブラリチェック、コードメトリクス、アクセシビリティチェック |
| 運用確認・リハーサル | 本番運用や障害対応に近い確認 | バックアップ、リストア、監視通知、リリース手順、ロールバック、データ移行 |

## 品質特性別の主な候補

| 品質特性 | 推奨されやすい確認方法 |
|---|---|
| 性能効率性 | 性能テスト、負荷テスト、計測ツール、定点観測 |
| 互換性 | 環境別テスト、外部IF確認、ファイル入出力確認、回帰テスト |
| 使用性 | シナリオテスト、ユーザビリティレビュー、アクセシビリティチェック |
| 信頼性 | 障害復旧テスト、例外系テスト、運用リハーサル、監視確認 |
| セキュリティ | 権限テスト、設計レビュー、脆弱性スキャン、ログ・監査証跡確認 |
| 保守性 | 設計レビュー、ログ確認、静的解析、運用手順レビュー |
| 移植性 | インストール、アップデート、移行、環境差分確認 |

## 選択時の注意

- 専用非機能テストにする前に、既存の機能テスト、レビュー、静的解析へ組み込めるか確認する。
- 主観判断が必要な使用性や保守性は、測定値だけで合格扱いにしない。
- 運用手順や復旧判断は、テスト実行だけでなく証跡とレビューを組み合わせる。
