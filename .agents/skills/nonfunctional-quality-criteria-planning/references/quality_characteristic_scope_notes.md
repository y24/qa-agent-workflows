# 品質特性スコープノート

## 対象品質特性

| 品質特性 | このskillでの扱い | 例 |
|---|---|---|
| 性能効率性 | 時間、資源、容量に関する品質 | 応答時間、処理時間、同時利用、データ量増加 |
| 互換性 | 他システム、環境、データ形式との共存・相互運用 | ブラウザ差異、OS差異、外部IF、Excel出力 |
| 使用性 | 利用者が理解し、誤り少なく操作できる品質 | 文言、操作導線、エラー時の復帰、アクセシビリティ |
| 信頼性 | 必要な時に正しく継続・復旧できる品質 | 障害時復旧、例外処理、バックアップ、可用性 |
| セキュリティ | 認証、認可、機密性、完全性、監査性 | 権限外操作、ログ、情報漏えい、脆弱性 |
| 保守性 | 調査、修正、拡張、運用保守のしやすさ | ログ、設定分離、変更影響、監視、コード・設計の分かりやすさ |
| 移植性 | 別環境への導入、移行、更新のしやすさ | インストール、移行、クラウド・OS変更、バージョンアップ |

## 対象外・補足

| 項目 | 扱い |
|---|---|
| 機能適合性 | 原則対象外。機能仕様そのものの正しさは `$scenario-test-design` などで扱う。 |
| 安全性 | 原則対象外。ただし業務、法令、人身、環境に重大影響がある場合は補足事項として記録する。 |
| プロジェクトリスク | 期間、体制、予算、環境不足はテスト制約または確認事項に分ける。 |

## ISO/IEC 25010との関係

このskillでは、実務上扱いやすい7特性名を採用する。ISO/IEC 25010:2023の用語差分は厳密な規格解釈として扱わず、分類補助として使う。

| このskillの名称 | ISO/IEC 25010上の近い概念 | 備考 |
|---|---|---|
| 性能効率性 | Performance Efficiency | 対象 |
| 互換性 | Compatibility | 対象 |
| 使用性 | Usability / Interaction Capability | 実務用語として使用性を使う |
| 信頼性 | Reliability | 対象 |
| セキュリティ | Security | 対象 |
| 保守性 | Maintainability | 対象 |
| 移植性 | Portability / Flexibility | 実務用語として移植性を使う |

## 分類時の注意

- 複数特性にまたがる懸念は、主影響で分類し、副特性をメモする。
- 使用性と機能仕様が混ざる場合は、「利用者が迷う、誤る、復帰できない」品質問題に限定する。
- セキュリティと機能権限が混ざる場合は、権限外操作や監査性など品質上の影響を明示する。
- 保守性は、テスト実行で直接判定しにくい場合があるため、レビューやログ確認も選択肢にする。
