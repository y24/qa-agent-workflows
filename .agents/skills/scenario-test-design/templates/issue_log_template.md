# 確認事項・課題ログテンプレート

## 1. 目的

このテンプレートは、シナリオテスト設計の各stepで発生した確認事項、仕様不足、文書間の矛盾、判断待ち事項、対象外候補を一元管理するためのものである。

AIエージェントは、各stepの成果物に記載した確認事項を、必要に応じてこのログへ集約する。

このログは、後続stepで以下の目的に使用する。

- 未解決の確認事項を見落とさない
- 仕様不足を前提にした誤ったシナリオ化を防ぐ
- 対象外候補を勝手に復活させない
- ユーザーレビュー結果を後続stepに反映する
- テストケース化できない理由を追跡する

## 5. 種別の定義

| 種別 | 意味 |
|---|---|
| 不明 | 必要な情報が入力文書や前step成果物から判断できない |
| 矛盾 | 文書間、同一文書内、またはレビューコメントとの間で内容が矛盾している |
| 仕様不足 | シナリオ設計やテストケース化に必要な仕様が不足している |
| 判断待ち | ユーザーまたは関係者の判断が必要 |
| 対象外候補 | 今回のシナリオテスト対象外にする可能性がある |
| リスク | 業務上または品質上のリスクとして管理すべき事項 |
| 保留 | 現時点では結論を出さず、後続判断に回す事項 |

---

## 6. 状態の定義

| 状態 | 意味 |
|---|---|
| 未確認 | まだ回答・判断がない |
| 確認中 | ユーザーまたは関係者に確認中 |
| 解決 | 回答・判断が得られ、後続stepへ反映可能 |
| 保留 | 今回は判断しない、または後で判断する |
| 対象外 | 今回の対象外として扱う |
| 取下げ | 確認事項として扱う必要がなくなった |

---

## 7. 優先度の定義

| 優先度 | 意味 |
|---|---|
| 高 | 未解決のままだと業務フロー、シナリオ候補、テストケースの妥当性に大きく影響する |
| 中 | 後続stepで一部影響するが、仮置きや保留で進行可能 |
| 低 | 補足情報として有用だが、主要成果物への影響は小さい |

## 16. AIエージェントへの運用ルール

AIエージェントは、このログを参照する際、以下を守ること。

- `未確認` の事項を、解決済みとして扱わない
- `保留` の事項を、勝手にシナリオ化・テストケース化しない
- `対象外` の事項を、後続stepで勝手に復活させない
- `矛盾` がある場合、どちらかを勝手に採用しない
- `仕様不足` がある場合、期待結果や操作手順を想像で補完しない
- 回答履歴がある場合は、回答内容を優先して反映する
- 未解決事項がP0/P1シナリオに影響する場合は、成果物内で明示する
- 後続stepへの反映指示がある場合は、それに従う
- 判断に迷う場合は、新たな確認事項として追加する
