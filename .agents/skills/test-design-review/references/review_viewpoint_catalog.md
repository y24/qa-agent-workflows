# Review Viewpoint Catalog

このカタログは、Step 03で観点不足や過剰を確認するための補助資料である。対象システム、テストレベル、レビュー範囲に応じて該当しそうな観点だけを使う。全項目を必須要求にしない。

## 機能・業務

| 観点 | 確認内容 |
|---|---|
| 正常系 | 主要な業務フロー、標準的な入力、代表的な利用条件が確認されているか |
| 異常系 | 入力エラー、業務エラー、権限エラー、外部連携エラーが考慮されているか |
| 境界値 | 数値、日付、桁数、件数、期間、範囲の境界が確認されているか |
| 業務ルール | 計算、判定、締め、承認、取消、例外条件が仕様に沿っているか |
| 状態遷移 | 登録、更新、取消、承認、差戻し、削除などの状態変化が確認されているか |
| 権限 | ロール、所属、所有者、操作権限、参照範囲の差異が確認されているか |

## データ・入出力

| 観点 | 確認内容 |
|---|---|
| 入力チェック | 必須、形式、桁数、範囲、禁止文字、組み合わせが分かれているか |
| データ整合性 | 画面、DB、帳票、連携先、履歴の整合性が確認されているか |
| 検索・一覧 | 検索条件、ソート、ページング、0件、複数件、表示項目が確認されているか |
| 帳票・出力 | 出力条件、項目、形式、計算結果、ファイル名、文字化けが確認されているか |
| 外部連携 | 正常応答、異常応答、タイムアウト、再送、冪等性が必要に応じて確認されているか |

## 影響・非機能

| 観点 | 確認内容 |
|---|---|
| 回帰影響 | 変更による既存機能、周辺機能、共通部品への影響が確認されているか |
| 過去障害 | 類似不具合、再発防止観点、障害多発箇所が反映されているか |
| 性能・容量 | 件数、応答時間、同時実行、大量データの懸念が必要に応じて扱われているか |
| 操作性 | 誤操作、戻る、二重送信、メッセージ、入力補助が必要に応じて確認されているか |
| 運用・保守 | ログ、監査証跡、リカバリ、運用手順、監視が必要に応じて確認されているか |
| セキュリティ | 認証、認可、情報漏えい、監査証跡、不正入力が必要に応じて確認されているか |
